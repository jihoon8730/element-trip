## 왜 만들었나

기존 여행 앱은 인기 명소 리스트를 보여줄 뿐, **나에게 맞는 여행**을 설계해주지 않습니다.

Elementrip은 사용자의 사주(생년월일시)를 오행(木火土金水) 분포로 분석하고, 한국관광공사 OpenAPI의 관광지 데이터를 같은 오행 속성으로 레이블링해 **궁합 점수** 기반의 맞춤 여행지를 추천합니다. "부족한 기운을 채우는 여행"이라는 웰니스 관점의 새로운 여행 프레임을 제시합니다.

## 아키텍처

```
[사용자 입력]
 생년월일시
      │
      ▼
[사주 분석 엔진]
 오행(木火土金水) 비율 산출
      │
      ├──────────────────────────────────┐
      ▼                                  ▼
[관광지 매칭 DB]                [실시간 날씨 보정]
 LLM이 사전 레이블링한            기상청 API →
 관광공사 OpenAPI 데이터           날씨 기반 가중치 조정
      │                                  │
      └──────────┬───────────────────────┘
                 ▼
         [궁합 점수 계산]
          유저 오행 ↔ 관광지 오행 대조
                 │
                 ▼
         [개인화 추천 결과]
          부족 기운 기반 맞춤 여행지
```

**매칭 엔진 설계 포인트**

관광지 텍스트(설명, 태그)를 LLM으로 배치 분석해 오행 속성 레이블을 사전 DB화합니다. (주 1회 갱신)
실시간 추천 시에는 유저 사주 분석값과 레이블 DB를 대조하기만 하므로 LLM 호출 없이 빠르게 응답합니다.

## 기술 스택

| 구분       | 기술                              | 선택 이유                                                                |
| ---------- | --------------------------------- | ------------------------------------------------------------------------ |
| 프레임워크 | Expo SDK 56                       | 단일 코드베이스로 iOS/Android 대응, EAS Build로 공모전 일정 내 빠른 배포 |
| 라우팅     | Expo Router (파일 기반)           | `app/` 디렉토리 구조가 곧 라우팅, 네비게이션 코드 최소화                 |
| 탭바       | NativeTabs (unstable-native-tabs) | iOS 26 Liquid Glass 효과를 네이티브 수준으로 구현                        |
| 애니메이션 | Reanimated 4 + Worklets           | UI 스레드 독립 실행으로 전환 애니메이션 60fps 유지                       |
| 외부 API   | 한국관광공사 OpenAPI, 기상청 API  | 관광지 메타데이터 + 실시간 날씨 보정                                     |

## 프로젝트 구조

```
src/
└── app/
    ├── _layout.tsx          # 루트 Stack 레이아웃
    └── (tabs)/
        ├── _layout.tsx      # NativeTabs 탭바 구성
        ├── index.tsx        # 나의 사주 (홈)
        ├── recommend.tsx    # 맞춤 추천 스팟
        ├── course.tsx       # 행운의 여행길
        └── archive.tsx      # 기운 아카이브
```

## 시작하기

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npx expo start
```

실행 후 앱 열기:

- iOS 시뮬레이터: `i`
- Android 에뮬레이터: `a`
- 실기기: Expo Go 앱으로 QR 스캔
