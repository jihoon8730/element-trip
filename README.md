> "내 안의 사주와 지역의 기운이 만나는 순간, 데이터로 설계하는 가장 나다운 여행"

## 왜 만들었나

기존 여행 앱은 인기 명소 리스트를 보여줄 뿐, **나에게 맞는 여행**을 설계해주지 않습니다.

Elementrip은 사용자의 사주(생년월일시)를 오행(木火土金水) 분포로 분석하고, 한국관광공사 OpenAPI의 관광지 데이터를 같은 오행 속성으로 레이블링해 **궁합 점수** 기반의 맞춤 여행지를 추천합니다. "부족한 기운을 채우는 여행"이라는 웰니스 관점의 새로운 여행 프레임을 제시합니다.

## 아키텍처

```
[사용자 입력]
 생년월일시
      │
      ▼
[사주 분석 엔진]
 오행(木火土金水) 비율 산출
      │
      ├─────────────────────────────────┐
      ▼                                 ▼
[관광지 매칭 DB]               [실시간 날씨 보정]
 LLM이 사전 레이블링한           기상청 API →
 관광공사 OpenAPI 데이터          날씨 기반 가중치 조정
      │                                 │
      └──────────┬──────────────────────┘
                 ▼
         [궁합 점수 계산]
          유저 오행 ↔ 관광지 오행 대조
                 │
                 ▼
         [개인화 UI 렌더링]
          부족 기운 컬러로 포인트 컬러 런타임 변경
```

**매칭 엔진 설계 포인트**

관광지 텍스트(설명, 태그)를 LLM으로 배치 분석해 오행 속성 레이블을 사전 DB화합니다. (주 1회 갱신)  
실시간 추천 시에는 유저 사주 분석값과 레이블 DB를 대조하기만 하므로 LLM 호출 없이 빠르게 응답합니다.

## 기술 스택

| 구분        | 기술                              | 선택 이유                                                                          |
| ----------- | --------------------------------- | ---------------------------------------------------------------------------------- |
| 프레임워크  | Expo SDK 56                       | 단일 코드베이스로 iOS/Android 대응, EAS Build로 공모전 일정 내 빠른 배포           |
| 라우팅      | Expo Router (파일 기반)           | `app/` 디렉토리 구조가 곧 라우팅이 되어 네비게이션 코드 최소화                     |
| 탭바        | NativeTabs (unstable-native-tabs) | iOS 17+ Liquid Glass 효과를 네이티브 수준으로 구현, UITabBarController 직접 바인딩 |
| 애니메이션  | Reanimated 4 + Worklets           | JS 스레드와 독립된 UI 스레드 실행으로 온보딩 전환 애니메이션 60fps 유지            |
| 글래스 효과 | expo-glass-effect                 | 팝업/오버레이 글래스모피즘 구현, 강한 딤드 오버레이 배제                           |
| 외부 API    | 한국관광공사 OpenAPI, 기상청 API  | 관광지 메타데이터 + 실시간 날씨 보정                                               |
| 디자인 토큰 | `src/constants/tokens.ts`         | 5기운 팔레트를 토큰으로 선언, 런타임 포인트 컬러 리스킨 지원                       |

## 디자인 시스템

**톤앤매너: Neo-Spiritual Wellness**  
무속·한자 그래픽을 배제하고 현대적 명상/웰니스 무드로 구성했습니다.

**5기운 팔레트** — 유저의 부족 기운에 따라 포인트 컬러가 런타임 변경됩니다.

| 기운    | 한국명    | 메인      | 그라데이션 |
| ------- | --------- | --------- | ---------- |
| `wood`  | 나무 (木) | `#E2F0D9` | `#C5E1A5`  |
| `fire`  | 불 (火)   | `#FCE4D6` | `#F8BBD0`  |
| `earth` | 흙 (土)   | `#FFF2CC` | `#FFE0B2`  |
| `metal` | 쇠 (金)   | `#E1D5E7` | `#D1C4E9`  |
| `water` | 물 (水)   | `#DAE8FC` | `#B3E5FC`  |

컬러는 항상 `tokens.ts`의 변수로 참조하며 하드코딩하지 않습니다.

## 프로젝트 구조

```
src/
├── app/
│   ├── _layout.tsx          # 루트 Stack 레이아웃
│   └── (tabs)/
│       ├── _layout.tsx      # NativeTabs 탭바 구성
│       ├── index.tsx        # 나의 사주 (홈)
│       ├── recommend.tsx    # 맞춤 추천 스팟
│       ├── course.tsx       # 행운의 여행길
│       └── archive.tsx      # 기운 아카이브
└── constants/
    └── tokens.ts            # 디자인 토큰 (컬러, 반경, 그림자)
```

## 시작하기

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npx expo start
```

실행 후 앱 열기:

- iOS 시뮬레이터: `i`
- Android 에뮬레이터: `a`
- 실기기: Expo Go 앱으로 QR 스캔

### 환경 변수

루트에 `.env` 파일을 생성하세요.

```env
EXPO_PUBLIC_KTO_API_KEY=   # 한국관광공사 OpenAPI 키
EXPO_PUBLIC_KMA_API_KEY=   # 기상청 API 키
```
